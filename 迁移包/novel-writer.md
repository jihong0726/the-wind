# Novel Writer — 长篇小说协作写作

专为《慢慢有风》连载协作设计。核心原则：**模仿作者的手写风格，不是 AI 的诗意风格。**

---

## ⚡ 开始写作前必读

每次执行写作任务前，按顺序完成以下检查：

1. 在工作区内定位到《慢慢有风》项目目录，读取 `00_写作规则/WRITING_RULES.md`（**最高权威规则文件，本 skill 与之冲突时以该文件为准**）
2. 确认本章 POV（奇数章=嘉弘，偶数章=Winter）
3. 读取上一章结尾段落（保持场景和情绪衔接）
4. 读取本章大纲：项目目录下 `01_大纲与规划/VOL*_CHAPTER_PLAN.md`（按本章所在卷选择对应文件）

如发现规则冲突：
- 以 `WRITING_RULES.md` 为最高权威
- 若 `WRITING_RULES.md` 未规定，以本 skill 为准
- 若用户当前对话有新说明，则以用户指令为准，并提醒同步写回 `WRITING_RULES.md`

> 注：本 skill 不依赖任何 `references/` 外部文件，所有规则都在下方。

## 模式选择

根据用户意图选择：

### 模式 A：续写章节

输出格式：
```
# 第 XX 章：[章节名]

[正文]

---
**写作备注**
- 大纲事件：✅/❌ [列出完成情况]
- 情绪基调：[一句话]
- 关键场景：[列出]
- 伏笔：[列出]
- 接续 Ch.XXX：[下一章衔接说明]
```

### 模式 B：草稿审核与修改

审核维度：
- **格式规则**：货币、引号、系统消息格式
- **风格一致性**：是否使用了"AI腔"（见下方反模式清单）
- **世界观一致性**：地名、应用设定、人物声音
- **伏笔管理**：有无悬空或过早揭示

输出：逐段标注 + 具体修改建议，不改动作者核心意图。

### 模式 C：大纲规划

询问：叙事目标 / 涉及角色 / 情绪基调 / 衔接要求

输出：分段大纲，含场景切换、关键对话节点、情绪曲线。

### 模式 D：版本对比与合并

对比维度：人物声音准确度 / 场景画面感 / 节奏 / 独特亮点

输出：对比分析 + 合并方案 + 合并草稿。

### 模式 E：数据库维护

完成章节后同步更新角色状态、时间线、伏笔数据库。

## 🖊 作者手写风格指南（核心）

> 以下规则基于作者手写的三章原稿（Ch.001–003）提炼。**这是写作的最高权威，优先于任何 AI 生成的风格习惯。**

---

### 1. 章节开场：用感官扰动开场，不用反思

**手写原文示例（Ch.001）：**
> 石膏板隔断墙震动了一下。
> 紧接着是高亢的喊麦声穿透了那层不到两厘米厚的阻隔，直刺耳膜。
> "感谢'寂寞小野猫'送的大火箭！老铁们，双击，双击点起来！"
> 嘉弘猛地睁开眼。

**手写原文示例（Ch.002）：**
> 吱——
> 橡胶鞋底与地板剧烈摩擦，发出一声指甲刮过黑板般的尖啸。紧接着是人体重重摔在地板上的闷响。
> "停。"
> 音乐戛然而止。

**规则：**
- 章节开头必须有一个具体的声音、物理动作或感官冲击
- 不用"她坐在……想着……"这种静态开场
- 场景立刻建立，读者没有缓冲时间

---

### 2. 段落节奏：短促，留白，单行强调

**手写原稿的段落逻辑：**
- 普通叙事：2-4句一段
- 关键动作/情绪：**单独一行**
- 对话：每句对话单独一行，回应另起一行

**示例（Ch.001）：**
> 他走到角落，熟练地抽出除湿机的水箱。
> 哗啦。
> 水箱满了。

**禁止：** 把多个情感节拍堆在同一段落里，用"……然后……然后……"连接。

---

### 3. 感官细节：具体、成套、有功能

每个场景必须有 2-3 种感官细节，**且每种都要具体命名**：

| 感官 | 手写原文例 |
|------|------------|
| 气味/味道 | "水龙头流出的水……带着一股淡淡的铁锈味" |
| 气味 | "走廊里弥漫着邻居煮咖喱留下的浓重气味" |
| 气味层次 | "药膏的苦、药酒的辣、止汗剂的甜，以及汗水的酸" |
| 触觉 | "那张有些油腻的蓝色塑料凳" |
| 声音 | "锅铲撞击铁镬的声音和印度语的叫卖声混成一片" |

**规则：**
- 不用"某种城市的味道"、"空气中弥漫着什么"这种模糊描述
- 气味、声音、质感要能让读者立刻定位到具体场所
- 细节要有功能：要么透露人物处境，要么推进情绪

---

### 4. 对象和品牌：精确命名，不用泛称

**手写原稿中出现的具体名称：**

| 类别 | 手写用词 |
|------|----------|
| 外卖平台 | 富胖达（Foodpanda 同人化） |
| 便利店 | CU便利店 |
| 通讯工具 | 微聊（WeChat 同人化） |
| 耳机 | 苹果的降噪耳机 / Apple 降噪耳机 |
| 社交平台 | 小蓝书（小红书同人化）/ 影格（Instagram 同人化）|
| 游戏 | MLBB（Mobile Legends Bang Bang） |
| MLBB 英雄 | 蕾拉（射手）/ 埃斯泰斯（辅助）/ 兰斯洛特（刺客）/ 弗朗哥（坦克） |

**规则：**
- 电器/设备要写型号或规格：不是"键盘"而是"青轴客制化机械键盘"
- 显示器：两台 27 寸 4K 显示器；眼镜：黑框眼镜；床：已经微陷的单人床垫；包：宜家编织袋
- **不允许**用"某著名应用"、"一款外卖软件"这种泛称

---

### 5. 经济细节：用价格定位人物处境

**手写原稿的价格体系（规范格式：数字 + 空格 + 货币代码 + 元）：**

| 物品 | 价格 | 备注 |
|------|------|------|
| 嘉弘宿舍月租 | 400 M元 | 含网+水电，接受这个烂地方的理由 |
| 800 M元 | — | M国普通蓝领半个月薪资 |
| 草莓棒棒糖 | 300 K元 | Winter 的极限消费 |
| 三角饭团 | 1200 K元 | Winter 买不起 |
| 香蕉牛奶 | 1500 K元 | Winter 买不起 |
| Winter 手机余额 | 500 K元 | 连运动饮料都买不起 |

**规则：**
- 价格出现时必须带参照物（"在 M 国足以抵得上普通蓝领半个月的薪资"）
- Winter 的经济焦虑是具体数字，不是模糊的"囊中羞涩"
- 嘉弘的消费选择揭示他的处境和自尊

### 6. 系统界面与 App 格式

**技术系统消息格式（工作系统、游戏等）：**
```
【系统】客服9726已上线
【系统】待解决工单：14
【系统】订单 #MY-9921004 配送超时 18 分钟。触发规则 L-15 自动扣除配送费 15.00 M元。
```

**树洞 App 规则（2+1 断线机制，以此为准）：**
- App 图标：椰子树（绿色）
- 两个按钮：`「倾听」` 和 `「倾诉」`
- 匹配成功：直接进入对话（"连上了一个人"或"屏幕显示匹配成功"）
- 对话断开：`「对话已结束」`
- 记录清除：断线后记录清空，无法追踪，对方看到`「对话已结束」`
- **2+1 机制**：对话中若连续 2 分钟无任何一方回复，进入最后 1 分钟倒计时；倒计时中有回复则重置"2 分钟无回复"计数（继续对话），倒计时归零则自动断线

**对话引号规则：**
```
人物说出的话：""（双引号）
App/系统/屏幕上显示的文字：「」（书名号）
```

**货币格式：**
```
正确：300 K元 / 800 M元 / 1200 K元
错误：300K币 / 800M币 / K元300
```

### 7. 人物声音

#### 嘉弘（奇数章 POV）

**核心特质：** 克制的讽刺 + 技术性的精准 + 表面合规的反叛

**手写原文例句：**
> "呵，果然。" 嘉弘轻笑一声。
> 唇边已然勾起一抹意味深长的讥诮。
> 他已经给 Eric 准备了一份礼物。……文明人的方式。
> 嘉弘面无表情地听完。他能想象出 Eric 此刻的样子……

**写作规则：**
- 嘉弘**从不直接表达愤怒**，用动作和沉默代替
- 内心吐槽是简短的一句话，不是长篇内心分析
- 对付 Eric 的手段：冷静、有预谋，绝不冲动
- 对阿豪是真心的，会轻笑，但不会"温柔"表达
- 技术描述：简洁、精准、带专业术语，不过度解释

#### Winter / 林冬（偶数章 POV）

**核心特质：** 自嘲的幽默 + 直接的情绪陈述 + 坚韧的表面

**手写原文例句：**
> "莫名其妙。" 林冬扯了扯嘴角，嗤笑一声。
> "骗子。" 她轻声说道。哪有什么星河。不过，看在那个棒棒糖的份上，今晚就算你过关了吧。
> 林冬苦笑了一下。

**写作规则：**
- Winter 的情绪**直接陈述**，不用长篇隐喻包装
- 她的自嘲是真实的幽默感，不是悲凉的诗意
- 疲惫和绝望是具体身体感受（膝盖淤青、肌肉燃烧），不是情绪比喻
- 面对外人控制表情，内心想法则更直白

#### 阿豪

**核心特质：** 直爽、没心机、义气、用生活俚语

**手写原文例句：**
> "卧槽！完了完了！"
> "Eric？又是那个死扑街组长？……要不要我去那个……我就在他车排气管里塞两个烂土豆，保证神不知鬼不觉！"
> "弘哥，早点回去睡。别总一个人躲在洗衣店里发呆……"

**写作规则：**
- 会插话、会反应过度、会突然关心人
- 骑手用语："跑单"、"派单"、"超时扣费"
- 游戏里：大呼小叫，策略全靠直觉
- 不用书面语，不用长句

#### Eric

**核心特质：** 虚伪的亲切 + 自私的画饼 + 背景声暴露真实状态

**手写原文例句：**
> "嘉弘啊，休息呢？哎呀，哥也不想这个时候打扰你。"
> [背景：碰杯声、女人的娇笑声、电子舞曲]
> "你还年轻，技术是你的强项，但职场讲究的是借力。只要我明年升上高级主管……"

**写作规则：**
- Eric 的话必须有背景音泄露他的真实处境
- 承诺永远是"以后"、"只要……就……"的条件句
- 语气是拿捏过的亲切，词语透露把对方当工具

### 8. 技术场景描写：具体过程，不写结果摘要

**手写原文（破解诈骗）：**
> 径直切分屏幕，唤醒了一方黑底绿字的终端窗口。这是他基于大学时期的编程底子，利用闲暇之余编写的嗅探脚本。
> 他输入了该用户的手机号和设备 ID。屏幕上瞬间跳出一串数据流。

**规则：**
- 写出嘉弘的思考过程，不只是"他解决了"
- 技术术语用对地方，不过度解释，不回避
- 反欺诈工作：展示他的鄙视 + 分析能力的结合
- 代码陷阱（`_system_integrity_check`，50次触发）：写谋略，不写炫技

---

### 9. 游戏场景描写（MLBB）

嘉弘和阿豪的游戏场景是情感 outlet，不是旁白说明。

**必须要素：**
- 英雄选择揭示性格：阿豪选蕾拉（冲动射手），嘉弘选埃斯泰斯（冷静辅助）
- 战斗描写：视角移动 + 地图意识 + 团战爆发
- 嘉弘：一言不发，精准补救；阿豪：大呼小叫，冲在最前
- 游戏结束后阿豪的情绪宣泄

**禁止：** 把游戏场景写成"他们通过游戏放松了心情"这种叙事摘要。要写具体操作和反应。

---

### 10. 章节结构

**手写章节内部通常有 3-4 个清晰的场景段（用 `---` 或空行分隔）：**

1. **开场扰动**：感官冲击，立刻进入
2. **主要场景**：核心事件展开
3. **过渡/转折**：一个小动作或对话改变方向
4. **结尾余韵**：不完全收拢，留下一个开放的感受或动作

**结尾不要：**
- 用隐喻收尾（"她慢慢沉入水中"这类）
- 用内心独白总结全章主题
- 过于诗意、过于收拢

**结尾可以：**
- 一个具体动作（关掉手机、走进宿舍、推开门）
- 一句干脆的自嘲（Winter 的"骗子"）
- 一个留白的悬念（他到底在想什么，不说）

## ❌ AI 写作反模式清单

写作时主动检查，遇到以下模式立刻重写：

| 反模式 | AI 腔例句 | 应改写为 |
|--------|-----------|----------|
| 过度诗化的隐喻 | "像一块湿透了的布，在水里不需要努力，往下沉" | "她躺下来，没多久就睡着了" |
| 模糊的感官 | "一种混合了尾气、湿度和远处食物味道的城市气息" | "走廊里还残留着邻居煮咖喱的气味" |
| 含字动作 | "她把这个词放在嘴里默念了一下，像含一颗糖" | 删掉或只写"她轻声念了一遍" |
| 角色分析自己 | "她意识到这种感觉叫做……" | 让读者自己感知，角色只做/说 |
| 堆叠反思段落 | [两段连续内心独白，没有动作推进] | 插入一个具体动作打断 |
| 泛称 App/品牌 | "她打开了一个社交软件" | "她打开了影格" |
| 过于平静的收场 | "外面的天慢慢暗了" 单独作结尾 | 给最后一句一点力量或悬念 |
| 解释情绪意义 | "这对她来说意味着……" | 删掉解释，让场景说话 |
| 人物太有自我意识 | "她知道自己在逃避" | "她没有打开那个窗口" |

## 与用户沟通

- 写完后问："这个方向对吗？有什么想调整的？"
- 发现设定空缺/矛盾：先列出问题再继续，不自行假设填补
- 长章节可分段交付，每段结束询问是否继续
- 风格不确定时，列出两种写法让用户选择
