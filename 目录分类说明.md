# 目录分类说明

本文件记录仓库内容的分类逻辑，方便未来自己或协作者快速定位。

---

## 一、按前缀分类

| 前缀 | 含义 | 示例 |
|---|---|---|
| `00_` | **全局配置档** — 整本小说都依赖，每次动笔都可能查 | state.json / 总体规划 / 风格守护机制 / 笑话分配表 |
| `01_` | **卷级规划** — 当前卷专用，卷与卷之间不交叉 | 卷一分镜 / 场景卡 |
| `02_` | **正文稿** — 已完稿章节 | 第 001 章 ~ 第 017 章 |
| `03_` | （预留） | 未来可能加 大纲分卷 Ⅱ-Ⅵ |
| `99_` | **归档 / 参考 / 废弃** — 不影响当前稿的历史内容 | 参考资料 / 旧版 / 废纸篓（`.gitignore`） |

---

## 二、文件作用速查

### 全局配置（00_）
| 文件 | 查什么 | 写新稿前要读 |
|---|---|---|
| `00_state.json` | 当前时间、物件位置、人物状态、AI 指纹历史、笑话用过的 | ⭐ 每次都读 |
| `00_总体规划.md` | 六卷骨架、决策登记、风险清单、待定项 | 涉及跨卷伏笔时读 |
| `00_风格守护机制.md` | 字数硬标准、禁用词、AI 指纹 11 类、Seba 手法清单 | ⭐ 每次都过 checklist |
| `00_笑话分配表.md` | 角色笑话归属、禁用清单、密度规则、已用笑话 | 计划加笑话点时读 |

### 卷级规划（01_）
| 文件 | 查什么 |
|---|---|
| `01_卷一分镜_111章.md` | 卷 Ⅰ 111 章分镜 + V3 实际落地对照 |
| `01_场景卡_Ch004-010.md` | 早期场景卡（已被 state.json 覆盖，留作历史） |

### 正文稿（02_）
文件名规则：`第XXX章_<10字内短标题>.md`
每章尾部含"写作备注"：大纲事件 / Seba 注入点 / 物件锚点 / 伏笔 / 字数 / AI 指纹

### 归档（99_）
| 子目录 | 内容 | 是否提交 |
|---|---|---|
| `99_参考资料/` | 素材库、备份 skill、旧根目录文件 | ✅ 提交（历史保留） |
| `99_归档_旧版/` | 旧版 100 章草稿（canon 已推翻） | ✅ 提交（作为参照） |
| `99_废纸篓/` | 废弃临时文件 | ❌ 不提交（`.gitignore`） |

## 三、找稿流程

**我想写下一章：**
1. 读 `00_state.json` → 确认时间线、物件、人物位置
2. 读 `01_卷一分镜_111章.md` → 确认本章大纲事件
3. 读 `00_风格守护机制.md` → 过章前 checklist
4. 读上一章末尾（`02_正文/第XXX章_*.md`）→ 确保接续感
5. 动笔

**我想查某个物件 / 角色的当前状态：**
- 第一站：`00_state.json` → `characters` / `active_plotlines`
- 第二站：`02_正文/` 最近的 2-3 章写作备注

**我想查某个笑话有没有用过：**
- `00_state.json` → `jokes_used` 数组
- 交叉查 `00_笑话分配表.md` 第三节

**我想查 canon 原则：**
- `00_总体规划.md` → 二 / 三 / 四 / 五 节
- `00_state.json` 顶层的 `*_canon` 字段

## 四、版本控制原则

1. **每章完稿立刻提交**，commit message 格式：`Ch.XXX 完稿 · 字数 X / AI X.XX‰`
2. **state.json 每次随章节一起提交**，不单独 commit
3. **canon 推翻 / 重大决策**，单独一次 commit，message 写清楚版本号
4. **废纸篓不进版本历史**（已在 `.gitignore`）
5. **分支策略**：主干 `main`，实验性章节可开 `draft/Ch-XXX` 分支
