# 🔗 章节衔接与钩子法则 (Chapter Link Rules)

> **版本**: 2.0 (AI 执行版)
> **适用范围**: 所有章节结尾与下一章开头
> **核心逻辑**: Action (动作) > Explanation (解释); Visual (视觉) > Mental (心理)

## 1. 核心铁律 (The Iron Laws)
*所有生成内容必须通过以下检查，否则视为无效。*

### 1.1 视角锚定律 (POV Anchoring)
* **嘉弘线 (单数章)**：开头 100 字内必须出现 **[触觉/听觉]** 锚点。
    * *允许库*：机械键盘敲击声、除湿机水满提示音、Switch 按键回弹、外卖袋的塑料摩擦声、T 国雨夜的潮气。
    * *时间格式*：中文模糊时间（例：凌晨两点半、外卖送达时）。
* **林冬线 (双数章)**：开头 100 字内必须出现 **[视觉/体感]** 锚点。
    * *允许库*：车窗防窥膜反光、耳返的异物感、练习室镜子里的倒影、镁光灯残影、冰美式杯壁的水珠。
    * *时间格式*：数字精确时间（例：02:30、倒计时 05:00）。

### 1.2 树洞“屏幕物理”律 (Screen Physics)
* **绝对禁止**：描写屏幕以外的感知（如：“仿佛听到对方叹气”、“感觉到对方在笑”）。
* **仅允许状态**：
    * `对方正在输入...` (Typing ghost)
    * `对方无反应` (No response)
    * `已断开` (Disconnected)
    * `[系统] 匹配中...` (Matching)
* **时效性**：严格执行 **2+1 规则**（2分钟无反应 -> 1分钟倒数 -> 强制断开）。

### 1.3 拒绝总结律 (No Summarizing)
* **章尾禁止**：定论性语句、情感升华、上帝视角总结。
    * ❌ *两人终于明白了彼此的心意。*
    * ❌ *这注定是一个不眠之夜。*
* **章尾必须**：落在**动作**或**状态**上。
    * ✅ *他把手机扣在桌面上。*
    * ✅ *屏幕上的光标闪烁了最后一下，熄灭。*

## 2. 钩子类型与配对矩阵 (Hook Matrix)

### 2.1 同态接钩 (State Continuity)
*用于同一场景延续，或强调两人动作的同步性。*

| 章尾动作 (上一章) | 章头动作 (下一章) | 逻辑关系 |
| :--- | :--- | :--- |
| **嘉弘**：点下退款按钮，弹窗消失。 | **林冬**：点下发送按钮，屏幕显示“已发送”。 | **动作同步** (点击) |
| **林冬**：看着车窗外的雨发呆。 | **嘉弘**：听着窗外暴雨打在遮雨棚的声音。 | **环境共振** (雨) |
| **树洞**：显示 `对方无反应`。 | **树洞**：倒计时结束，跳出 `已断开`。 | **时间流逝** |

### 2.2 异态接钩 (State Contrast)
*用于切换视角时的身份反差，制造张力。*

| 章尾动作 (上一章) | 章头动作 (下一章) | 逻辑关系 |
| :--- | :--- | :--- |
| **嘉弘**：摘下耳机，世界瞬间安静。 | **林冬**：戴上耳返，噪音瞬间炸裂。 | **静 vs 噪** |
| **嘉弘**：关掉台灯，房间陷入黑暗。 | **林冬**：舞台灯光亮起，视线一片惨白。 | **暗 vs 亮** |
| **树洞**：输入框里打了一行字又删掉。 | **树洞**：盯着空白的对话框，指尖悬空。 | **删 vs 停** |
| **嘉弘**：躺进被窝，蜷缩。 | **林冬**：从保姆车里坐直，整理表情。 | **松 vs 紧** |

### 2.3 跨物理接钩 (Physical Handoff)
*利用 T 国/洗衣店/便利店作为中转站。*

| 章尾动作 (上一章) | 章头动作 (下一章) | 逻辑关系 |
| :--- | :--- | :--- |
| **嘉弘**：推开自助洗衣店的门，风铃响了一声。 | **林冬**：听见门口风铃响，下意识压低帽檐。 | **声音传递** |
| **林冬**：把硬币投进贩卖机，声音清脆。 | **嘉弘**：听着口袋里硬币碰撞的声音，数了数。 | **物件关联** |
| **嘉弘**：看着 7-11 关东煮冒出的热气。 | **林冬**：透过 7-11 的玻璃窗看到热气模糊了倒影。 | **视觉传递** |

## 3. 场景级钩子库 (Scenario Library)

### 3.1 树洞专用库 (The Hole)
* **[断开]**: 章尾显示 `已断开` -> 章头手指在 `重新匹配` 上悬停。
* **[沉默]**: 章尾 `对方正在输入...` 闪烁太久 -> 章头屏幕变暗，自动锁屏。
* **[彩蛋]**: 章尾嘉弘发送 *"星河璀璨..."* -> 章头林冬看着这行字，嘴角动了一下。

### 3.2 嘉弘工作库 (Work Side)
* **[升级]**: 章尾工单被标记为红色（升级） -> 章头林冬那边经纪人电话突然响起（危机同频）。
* **[下班]**: 章尾 00:00 登出系统，伸懒腰骨骼作响 -> 章头 00:00 行程表更新提醒弹出。
* **[生活]**: 章尾 Switch 没电黑屏 -> 章头林冬保姆车驶入隧道，视野变黑。

### 3.3 林冬行程库 (Idol Side)
* **[伪装]**: 章尾戴上黑色口罩，遮住半张脸 -> 章头嘉弘看见一个裹得严严实实的人进来。
* **[痛感]**: 章尾练舞导致膝盖磕在地上 -> 章头嘉弘觉得椅子太硬，换了个姿势。
* **[边界]**: 章尾经纪人收走手机 -> 章头树洞显示 `对方已下线`。

## 4. 熔断自检清单 (Stop & Check)
*AI 在生成每章结尾和开头后，必须执行以下检查：*

1.  **[ ] 上帝视角检查**：是否出现了“他不知道的是”、“与此同时”？ -> **有则重写**。
2.  **[ ] 感官越界检查**：是否在树洞里写了“听见对方笑”、“感到对方温柔”？ -> **有则重写**。
3.  **[ ] 锚点缺失检查**：开头 3 句内是否缺失了键盘/耳返/雨声等识别物？ -> **有则补全**。
4.  **[ ] 总结式结尾检查**：结尾是否是结论句？（如：这就是他们的初遇） -> **有则改为动作**。
